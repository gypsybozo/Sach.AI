# Sach.AI | Bias Mitigation in Indian Journalism

## Overview

This project aims to develop an automated bias mitigation pipeline tailored to Indian journalism. The system focuses on detecting and reducing caste, religion, political, and regional biases in news articles. By leveraging Natural Language Processing (NLP) techniques such as sentiment analysis and bias detection, the pipeline ensures fair and balanced reporting. The project includes a culturally contextualized dataset, explainable AI for transparency, and scalable clustering techniques to uncover bias patterns and optimize the framework for global applications.

## Key Features

- **Bias Detection**: Detects biases related to caste, religion, politics, and regions in news articles.
- **Culturally Contextualized Dataset**: Tailored dataset for Indian socio-cultural contexts to improve accuracy in bias detection.
- **Sentiment Analysis**: Identifies sentiment biases towards specific groups or topics.
- **Bias Quantification**: Provides bias scores to quantify the level of bias in content.
- **Bias Mitigation**: Transforms biased articles into neutral content for fair representation.
- **Explainable AI**: Ensures transparency by explaining how bias is detected and mitigated.
- **Scalable Clustering**: Uses clustering techniques to uncover bias patterns across large datasets.

## Dataset

The project uses several datasets, including:

1. **India News Headlines Dataset**: A collection of 3.8 million articles from *Times of India* (2001–2023). This dataset provides insights into societal trends and media priorities.
   

2. **Indian Express News Articles Dataset**: 20,000 news articles from *Indian Express* (2019–2020) representing a snapshot of Indian news during this period.
   

3. **HuffPost News Category Dataset**: 210,000 headlines from HuffPost (2012–2022), classified into 42 different categories.
4. **Web Scraped Data**: Publicly available data from Twitter and Reddit, focusing on user-generated content to analyze trends and sentiment.
